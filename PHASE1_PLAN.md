# ФАЗА 1 — Структура Godot-проекта «Красная Волна»

> Цель фазы 1 (3–4 недели): **серый прототип, одиночный.** 5 минут бега по серым кубам, с которых весело съезжать от первого лица; идут часы дня со сменой света; работает первая горка с физикой спуска, читающей вес. Сети, NPC, квестов в UI ещё нет — но архитектура заложена так, чтобы они встали без переписывания.

## 2. Автолоады (порядок важен)

1. **GameConstants** — чистые `const`, ни от кого не зависит.
2. **EventBus** — только `signal`-ы, развязывает системы (издатель/подписчик).
3. **Clock** — двигает `day_fraction` 0→1 за `RUN_LENGTH`, шлёт `phase_changed`, `scheduled_event(name)`. Разделяет мировое (×run_scale) и реакционное (реал) время.
4. **Hype** — на старте забега бросает Гул (ограничения §10), хранит `gul[slide_id]`, `day_slide`.
5. **WeightSystem** — состояние веса, API: `eat()`, `toilet()`, `add_run_distance()`, `speed_factor()`, `can_ride_extreme()`.
6. **Net** — стаб: эмулирует «сервер = это устройство», в фазе 3 станет настоящим авторитетом.

---

## 3. Архитектура сигналов (EventBus)

Системы не зовут друг друга напрямую — общаются через EventBus. Это критично для фазы 3 (сеть): сервер сможет перехватывать те же сигналы.

```
signal run_started()
signal run_planning_started()
signal phase_changed(phase: String)
signal scheduled_event(event: String)        # "show_1", "parade", "maintenance", "ballad"
signal slide_completed(player_id: int, slide_id: String)   # «достиг бассейна» (баг #21)
signal weight_changed(player_id: int, kg: float)
signal dizziness_changed(player_id: int, level: int)
signal zone_closed(player_id: int, zone: String)           # первопроходец (server-auth позже)
signal quest_progress(player_id: int, quest_id: String, done: bool)
signal ping_made(player_id: int, world_pos: Vector3, context: String)
```

## 5. Вехи фазы 1 (по неделям, ориентир)

- **Неделя 1.** Проект, git, автолоады GameConstants/EventBus/Clock. FPS-контроллер ходит по серой плоскости. Часы идут, свет меняется утро→вечер. Тест-сцена.
- **Неделя 2.** **SlideRail** — спуск по сплайну от первого лица, скорость = `WeightSystem.speed_factor()`. Это главный технический риск: если кататься не кайфово — стоп, чиним ощущения. Comfort-настройки.
- **Неделя 3.** Greybox-парк (3 зоны примитивами, центр, река-кольцо как путь), NavMesh. Hype бросает Гул на старте. WeightSystem полностью (еда/туалет/бег/лок) сHUD.
- **Неделя 4.** QuestGenerator как логика + `tests/quest_gen_check.gd` (перенос симуляции, прогон 200×, отчёт в консоль). Фаза планирования (пауза + заглушка карты). Вертикальный срез: 5 минут связного бега + катание.

**Критерий выхода из фазы 1:** даёшь сборку другу на 5 минут — он бегает, катается с одной горки, видит, как парк темнеет к вечеру, и это уже приятно.

## 7. Беклог (НЕ фаза 1 — записано, чтобы не забыть)

### Анти-эксплойт: нельзя скатываться, поднявшись по самой горке
**Проблема.** Если игрок может залезть вверх по жёлобу горки и съехать сверху, он
обходит очередь и правильный вход — это чит.

**Решение (2 уровня):**
1. **Мягкий барьер — поток воды.** Встречный поток по жёлобу достаточно сильный,
   чтобы не дать легко взобраться вверх (толкает игрока вниз). Сейчас вода на горке
   чисто декоративная (`slide_water.gdshader`) — позже она должна *физически толкать*
   против направления подъёма.
2. **Жёсткий барьер — штраф «хулиган».** Если игрок всё же упорно лезет вверх и
   въезжает не с легитимного входа (через очередь с правильной стороны):
   - штраф очков как хулигану (см. `SHAME = -2` в GameConstants),
   - **бан катания на ~2 минуты** (≈120 реальных сек; добавить константу, напр.
     `HOOLIGAN_RIDE_BAN`).

**Как отличать легитимный въезд от чита.** Старт катания засчитывается только из
зоны входа (`Mount`) с правильной стороны / после очереди; въезд из тела жёлоба или
против направления — нелегитимный → штраф + бан.

**Сетевая заметка.** Решение «легитимен ли въезд», штраф и бан — авторитет сервера
(как и Гул/квесты/вес), чтобы клиент не мог это подделать.

### Вес скрыт от игрока + пункты взвешивания
В играбельной версии игрок НЕ видит свой вес постоянно. На HUD — только **сколько
калорий потратил за забег**. Узнать точный вес можно на **пунктах взвешивания**,
расставленных по карте (взаимодействие у весов). Сейчас вес на HUD виден как
«(отладка)» — убрать перед релизом.

**Балансовая петля веса (зафиксировано v1.1).** Набор веса — только едой; еда нужна,
чтобы сбивать тошноту (головокружение) после горок — вот и компромисс. Переел →
≥91 кг → не пускают на экстрим (тост). Сбросить вес: бегать/прыгать (жгут калории
сильнее ходьбы/плавания) или туалет −3 кг, но туалет раз в 3 игровых часа (кулдаун) —
быстро не скинуть. Потолок веса 100 кг.

### NavMesh — отложен до NPC (фаза 2)
Плановый NavMesh «с фазы 1» осознанно отложен: без NPC-агентов он невидим и не
тестируется. Greybox-парк (`world/ParkBuilder.gd`) уже даёт плоскую проходимую
геометрию — `NavigationRegion3D` добавим и запечём, когда появятся NPC/очереди.

## 8. Прогресс (актуально)

**Фаза 1 — закрыта.** Вертикальный срез: планирование → день → финал → заново.
Автолоады, часы+свет (утро→красный закат), вес (калории/лок/туалет-кулдаун),
FPS+плавание (инерция, брызги, ленивая река-течение), горка по сплайну, HUD.

**Фаза 2 — почти закрыта.**
- ✅ Табло Гула, очереди+NPC, квест-трекинг + очки (главный квест, первопроходец зоны,
  без-очереди-бонус, штраф за Балладу), пинг (СКМ), карта M (карточки+метки) + миникарта.
- ⏳ Динамические очки гонок/доп-миссий — ждут своих систем (ф.4–5).

**Текущая архитектура горки/толпы (v2, наземная):**
- Горка стартует с земли и уходит в яму-бассейн (нет лестницы/телепорта).
- Горка САМА копает яму в общем полу (узел в группе `ground`) — сцены не координируют ямы.
- NPC = `CharacterBody3D` с коллизией (не призраки); очередь на земле; игрок учитывается
  снимком «сколько впереди»; слот занят весь цикл; светофор «вы следующий».

**Следующие шаги (по приоритету):**
1. **NavMesh + навигация NPC** — нужно ДО добавления остальных горок (иначе NPC застревают
   на дальних маршрутах). Только после этого расставлять 15 горок по зонам.
2. **Фаза 3 — сеть/кооп**: host-listen, авторитет сервера (часы, Гул, квесты, вес, очки,
   пинги), синк позиций. EventBus уже развязан под перехват сервером.
3. Контент-локации (кафе/отель/спа/театры) как POI на карте — ф.4–5.

**Валидация без редактора:** парсинг скриптов можно проверить headless —
`godot --headless --editor --quit --path <проект>` (Godot 4.6.3 Steam).
Рантайм-проверка: `--headless --fixed-fps 60 --quit-after N <scene>` (день авто-стартует
через 20с планирования). Новый `class_name` сперва требует `--editor` (регистрация).

### Доп. сделано (v1.2, автономно)
- Горка: **простой прямой скат** с земли в яму (труба совпадает с катанием); сама копает
  яму в полу (группа `ground`); **рабочая очередь** (передний едет по «свободно»),
  светофор, NPC с коллизией + анти-стак.
- Квесты: **личное доп-задание** (+5), **сложный режим** (штрафы −10/−5), пинги на карте.
- **POI:** пункты взвешивания (точный вес скрыт с HUD — только состояние+калории; число на
  весах) и **театры с расписанием шоу** (Clock `show_*` → посещение → квест «Театры» реальный).
- Карта/миникарта рисуют горки/зоны/реку/метки/пинги/весы/театры.

### v1.3 — большая территория + все атомы квестов реальны
- **Большой крытый парк**: купол, стены 3 этажа, панорама (Зеро/закат), 3 зоны, бассейн
  «Волны», река-капилляр **кольцом** с течением. Фундаменты: Рулетка/Рой/Комар.
- **Горки**: комплекс «Сердце» (klyk/plashch/krylo) + Дельта (Рой, гоночная) + Зеро (ruletka).
- **POI**: весы, театр(шоу), лавки(сувениры), Бард(фото), Fast Pass киоск.
- **Фоновая толпа** посетителей (Visitor) — оживляет территорию.
- **ВСЕ атомы квестов трекаются реально** (AUTO_AXES пуст): горки/еда/шоу/лавки/бард/
  круги-по-реке/без-очереди(fast pass)/гонка(Рой, призрак)/вес/голова. Любой квест проходим.

### v1.4 — умные NPC (3 типа поведения)
NPC-посетители горок теперь с типом (NPCAgent.Behavior):
- **TOUR** (голубые) — катаются подряд по ВСЕМ горкам;
- **POPULAR** (оранжевые) — только по популярным (Гул ≥ 60);
- **CASUAL** (зелёные) — 1–2 спуска, потом в театр или поплавать по реке, и снова.
Менеджер раздаёт смесь типов. Движение пока прямое + анти-стак (NavMesh сгладит — ниже).

### NavMesh — СДЕЛАНО (v1.5)
`NavigationRegion3D` + запекание по группе `navsource` (пол, стены, трубы, дно, пандусы)
на старте дня; `NavigationAgent3D` на NPC и Visitor ведёт их по навмешу (обход препятствий),
без навмеша — деградация в прямую + анти-стак. Проверено headless: 262 полигона, NPC ходят
и катаются. Тонкая настройка качества путей (cell_size/agent_radius/уклоны) — по визуалу.

### «Сердце» — многоуровневое здание (СДЕЛАНО, v1.6)
`HeartBuilding`: 3 уровня (вход/семейный/приключения), пандусы между этажами, стеклянные
перила, метки. Этажи отступают к югу, горки стартуют с северного края этажей и уходят вниз
без пересечений. `SlideRail.top_height` + `build_access=false` — горка стартует с этажа.
Горки: 2 семейные (2 этаж) + экстрим klyk (3 этаж). Проверено: NPC поднимаются по этажам
через навмеш и катаются со всех. Визуал/выравнивание — по скринам.

### Фаза 3 — кооп, первый слой (СДЕЛАНО, v1.7)
`Net`: реальный ENet `host()`/`join()` (порт 24545), peer-события, is_server/local_id из
multiplayer API. `CoopManager`: рассылает позицию локального игрока (rpc 20 Гц), показывает
аватары других (`RemoteAvatar`). Аддитивно — оффлайн = одиночная игра без изменений.
Запуск теста: `-- --host` / `-- --join`. Проверено двумя headless-инстансами: соединение,
обмен позициями, аватары созданы у обоих.

**Осталось в коопе:** server-authority синхрон состояния (Гул/квесты/часы/очки — чтобы
у всех один день), лобби-UI (хост/join в игре, не из cmdline), общие пинги, имена/цвета.

**Осталось крупное:** фундаменты Рулетка/Комар → объекты; синий/красный потоки концепта;
полировка по визуалу (выравнивание здания, пути NPC).
