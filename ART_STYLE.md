# Визуальный стиль «Красной Волны» — тун-шейдинг (Wind Waker)

Этот документ — точка входа в графику проекта. Стиль: **плоские чистые цвета +
ступенчатый свет + чёрный контур + мягкий свет/туман** (как в Zelda: Wind Waker).

**Главная идея разделения труда:**

| Что | Где делается | Почему |
|---|---|---|
| Форма (здания, декор) | Blender → `.glb` | красиво = нормальные модели с UV |
| Стиль (тун, контур, свет) | Godot (шейдеры + Environment) | «соус», объединяющий всё |
| Расстановка | редактор Godot (сцены/гизмо) | мышкой приятнее, чем константами |
| Геймплей (горки, NPC, лавки) | код | уже работает — не трогаем |

> **TL;DR для нетерпеливых:** всё уже настроено. Запусти проект — грейбокс уже
> в тун-стиле с обводкой. Чтобы добавить объект «в стиле», бери материал из
> фабрики: `node.material = Look.mat(Look.WOOD)`. Чтобы подвезти модель — клади
> `.glb` в `assets/models/` (см. [assets/models/README.md](assets/models/README.md)).

## 1. Что уже сделано (готовая база)

```
assets/shaders/
  toon.gdshader          ← непрозрачный тун-материал (основной)
  toon_alpha.gdshader    ← прозрачный тун (стекло, вода, подсветки зон)
  toon_light.gdshaderinc ← общая формула ступенчатого света (для двух шейдеров)
  outline.gdshader       ← чёрный контур («вывернутый корпус»)
  README.md              ← подробно про шейдеры
autoload/
  Look.gd                ← ФАБРИКА материалов + палитра (главный файл для тебя)
```

- **`Look.gd` зарегистрирован как autoload** (`project.godot`) — зовётся отовсюду как `Look`.
- **Все старые помощники материалов переведены на фабрику.** В `ParkBuilder.gd`,
  `HeartBuilding.gd`, `SlideRail.gd`, `StallPOI.gd`, `SpaPOI.gd`, `ItemShopPOI.gd`
  функции `_mat()/_make_material()` теперь возвращают `Look.mat(...)`. Поэтому весь
  существующий парк уже тун-стайл с обводкой — переписывать геометрию не нужно.
- **`WorldEnvironment`** в `world/ParkGreybox.tscn` настроен под тун: ровный
  голубой ambient, слабый glow, лёгкий туман, насыщенность чуть выше, «грязные»
  эффекты (SSAO/SSIL/SDFGI) выключены. Солнце с мягкой тенью (`shadow_blur`).
- Свет дня (`world/Lighting.gd`) крутит цвет/энергию по времени суток — оставлен.

## 2. Как этим пользоваться (главное)

Везде, где задаёшь цвет объекту, проси материал у фабрики `Look`:

```gdscript
# непрозрачный объект → тун + чёрный контур (автоматически)
csg_box.material = Look.mat(Color(0.6, 0.45, 0.32))
# или цветом из палитры по смыслу:
mesh.material_override = Look.mat(Look.WOOD)

# прозрачный (стекло/вода): alpha < 1 → фабрика сама берёт прозрачный шейдер,
# контур к таким НЕ вешается:
glass.material = Look.mat(Color(0.6, 0.8, 1.0, 0.25))

# светящийся (вывеска, неон, киоск, луч):
sign.material_override = Look.emissive(Color(0.2, 0.9, 0.9), 3.0)

# без контура (например, огромный пол): второй аргумент = false
floor.material = Look.mat(Look.STONE, false)
```

Сигнатуры (см. `autoload/Look.gd`):
- `Look.mat(color, outline := true, transparent := false) -> ShaderMaterial`
- `Look.emissive(color, energy := 2.0, outline := false) -> ShaderMaterial`

## 3. Крутилки стиля (где что менять)

| Хочу… | Где |
|---|---|
| Сильнее/слабее «мультяшность» света | `Look.BANDS` (2 = резко … 5 = мягче) |
| Толще/тоньше контур | `Look.OUTLINE_WIDTH` (метры) |
| Другой цвет контура | `Look.OUTLINE_COLOR` |
| Палитра парка | константы `SKY/WATER/WAVE/...` в `Look.gd` |
| Яркость/цвет ambient, туман, glow | нода `WorldEnvironment` в `ParkGreybox.tscn` |
| Цвет/жёсткость солнца по времени суток | `world/Lighting.gd` |

**Совет по палитре:** держись 8–12 базовых цветов из `Look.gd`. Единая палитра
важнее детализации — именно она «склеивает» картинку в один стиль.

## 4. Как добавить здание (Blender → Godot)

1. **Blender:** масштаб 1 unit = 1 метр, моделируй низкополи крупными формами.
2. Готово → `Ctrl+A → Apply Scale`, origin в основание здания.
3. **UV:** `U → Smart UV Project` (быстро) или вручную.
4. **Цвет:** либо Base Color в Principled BSDF (без текстуры — для тун норм),
   либо нарисуй текстуру (256–512 px) в Base Color.
5. **Экспорт:** `File → Export → glTF 2.0 (.glb)` → `assets/models/buildings/`.
6. **В Godot** `.glb` импортируется сам. Чтобы навесить тун:
   - простой путь: `mesh.material_override = Look.mat(Look.WOOD)` (или прямо в
     инспекторе перетащи/создай ShaderMaterial с `toon.gdshader`);
   - с текстурой модели: в импорт-настройках `.glb` сделай материалы
     «extractable», на surface поставь ShaderMaterial (`toon.gdshader`) и в его
     параметр `tex` подставь текстуру.
7. **Коллизия:** в Blender назови меш-коллайдер с суффиксом `-col` (Godot сделает
   StaticBody сам) ИЛИ в Godot добавь `StaticBody3D` + `CollisionShape3D`.

Не хочешь моделить — бери бесплатные паки (Kenney.nl, low-poly наборы) и вешай
свой `Look.mat(...)`. Очень быстрый путь к «красиво».

## 5. Расстановка по карте (конвенции проекта — ВАЖНО)

Два способа размещения, оба по назначению:

- **Сценами в редакторе (для декора/красоты, рекомендуется):** сделай сцену
  объекта, в `world/ParkGreybox.tscn` заведи узел-контейнер (напр. `Decor`),
  перетаскивай инстансы, двигай гизмо.
- **Кодом (для системных вещей, как сейчас):** зоны/лавки/фундаменты — константы
  в `ParkBuilder.gd` (`ZONES`, `FOUNDATIONS`, `SHOPS`, `FOOD_COURT`).

**Иначе сломаешь NPC/физику:**
- Пол и препятствия, по которым ходят NPC, добавляй в группу **`navsource`**
  (навмеш печётся на `run_started`).
- Коллизия: у CSG `use_collision = true`; у моделей — `StaticBody3D` +
  `CollisionShape3D` или суффикс `-col`.
- Вода/зоны/POI — это `Area3D` в своих группах (`water`, `slide`, `poi_*` …).
  Повторяй паттерн существующих POI (`ShopPOI`, `StallPOI`).

## 6. Горки (не моделить — рисовать путь)

Горка — это `SlideRail`: труба `CSGPolygon3D`, едущая вдоль `Path3D`. Чтобы задать
форму вручную:
1. У инстанса горки поставь `build_demo_curve = false`.
2. Добавь дочерний `Path3D` с именем `Path` (код подхватит его в `_setup_path()`).
3. Выдели `Path3D` → рисуй кривую гизмо в 3D. Труба и вода поедут по ней сами.

Материал трубы уже идёт через фабрику (`_make_material → Look.mat`) — тун + контур.

## 7. Рекомендуемый порядок работы

1. **Стиль уже стоит** — запусти и посмотри грейбокс в тун-стиле.
2. Подкрути палитру (`Look.gd`) и ambient/туман (`WorldEnvironment`) под вкус.
3. Замени самые заметные объекты (Сердце, лавки) на Blender-модели с `Look.mat`.
   Мелочь оставь на CSG.
4. Нарисуй пути горок вручную через `Path3D`.
5. Расставляй декор сценами в узле `Decor`, проверяя группу `navsource`.

## 8. Что НЕ переведено на фабрику (и почему)

Простые цветные материалы переведены. Осталось на `StandardMaterial3D` несколько
мест со **свечением/спецэффектами** — их можно перевести на `Look.emissive(...)`,
когда дойдут руки:
- NPC (`systems/npc/*`) — однотонные `albedo` (легко перевести на `Look.mat`);
- лучи/подсветки и киоски (`PlayerController.gd`, `FastPassKiosk.gd`,
  `StallPOI.gd`, `SlideRail.gd`) — эмиссия → `Look.emissive(...)`;
- пар в спа (`SpaPOI.gd`) и вода горки (`slide_water.gdshader`) — отдельные
  эффекты, осознанно оставлены как есть.
