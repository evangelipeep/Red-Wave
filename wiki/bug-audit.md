# Аудит логических дыр (баги #1–#24)

Ранний аудит дизайна: перечень логических дыр и решений по ним, сгруппированный по
темам. Ссылается на конкретные механики.

## Содержание

**Тайминг:** #1 дедлок [[weight-system|вес]]+экстрим (детур в туалет в SOLVABLE);
#2 головокружение-софтлок (учесть время на остывание); #3 коллизия техобслуживания
(квесты по тегам, проверять окно); #4 разделить мировое/реакционное время
([[day-cycle-and-time]]).

**Сеть** ([[coop-networking]]): #5 гонка за зону — только сервер по таймстемпу; #6
авторитет квестов на сервере; #7 вес сервер, бег санити-чек; #8 сглаживать позиции
для [[voice-chat|голоса]]; #9 чистый дисконнект; #10 host-преимущество принять.

**Генератор** ([[quest-system]]): #11 лимит попыток + [[fallback-bundles|безопасный fallback]]
(обязательно); #12 ep+время оба (валидировано); #13 утечка квеста через объявление
зоны — ОК; #14 одноразовость [[zone-pioneer-bonus|бонуса зоны]].

**Экономика/тело:** #15 SPEND_ALL vs покупки — ловит COMPATIBLE; #16 достижимость
[[side-missions|доп-миссий]] учитывает вес/головокружение.

**UX/карта** ([[markers-and-pings]]): #17 подпись метки опционально; #18 краевые
индикаторы на круглой миникарте; #19 решено — персональные метки + командный пинг;
#20 карта не ставит паузу (кроме фазы планирования).

**Определения:** #21 «прокатился» = достиг бассейна; #22 distinct vs repeat хранить
раздельно; #23 интервал [[bard-mascot|Барда]] findable; #24 тай-брейк определён
([[scoring]]).
